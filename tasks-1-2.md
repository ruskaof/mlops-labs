# MLOps-пайплайн для классификации мошеннических транзакций

## Описание проекта
**Задача:** Реализовать полный цикл разработки и внедрения модели машинного обучения для классификации мошеннических транзакций. Проект представляет собой единый, последовательно развивающийся репозиторий, охватывающий все этапы жизненного цикла ML-системы.

**Источник данных:** [Credit Card Fraud Detection](https://www.openml.org/search?type=data&id=43627&sort=runs&status=active) — классический датасет с сильным дисбалансом классов (мошеннические транзакции составляют малую долю от общего числа), что идеально подходит для имитации реальных задач.

**Стек технологий:**
- **Язык:** Python
- **Контейнеризация:** Docker
- **Версионирование данных:** DVC
- **Трекинг экспериментов:** MLflow
- **API:** FastAPI
- **Качество данных:** Great Expectations
- **CI/CD:** GitHub Actions

### Неделя 1: Жизненный цикл и Инициализация
На этой неделе закладывается фундамент проекта: настройка репозитория, первичный анализ данных и подготовка воспроизводимого окружения.

1. **Инициализация проекта**
   - Создать репозиторий на GitHub.
   - Инициализировать структуру проекта (рекомендуется использовать стандартный шаблон, например, `Cookiecutter Data Science`).

2. **Работа с данными (EDA)**
   - Скачать датасет из [источника](https://www.openml.org/data/download/22102452/dataset).
   - Провести базовый разведочный анализ данных (EDA) в Jupyter Notebook для понимания распределения признаков и степени дисбаланса классов.

3. **Определение целей**
   - Сформулировать бизнес-цели проекта.
   - Выбрать ключевые метрики успеха (основной фокус на **Recall** для класса Fraud, а также **F1-score**), так как важно не пропускать мошеннические операции.

4. **Управление зависимостями**
   - Настроить менеджер зависимостей (`Poetry` или `pip`) для жесткой фиксации версий библиотек.

5. **Контейнеризация** 
   - Написать `Dockerfile`, описывающий окружение, необходимое для запуска всех скриптов проекта.

6. **Тестирование окружения**
   - Проверить работоспособность: команды `docker build` и `docker run` должны успешно выполняться на локальной машине, создавая изолированную среду.

### Неделя 2: Версионирование данных и Трекинг экспериментов
На этой неделе осуществляется внедрение инструментов контроля версий данных и систем логирования экспериментов для обеспечения воспроизводимости.

1. **Инициализация DVC**
   - Инициализировать DVC в проекте.
   - Настроить удаленное хранилище (DVC Remote) — использовать S3 или локальную папку (как симуляцию S3).

2. **Контроль версий данных**
   - Добавить исходный датасет (`data/raw`) под контроль DVC.

3. **Версионирование подготовки данных**
   - Выполнить разделение данных на train/test и сохранить версии разделенных датасетов.

4. **Развертывание MLflow**
   - Поднять локальный сервер MLflow через Docker.

5. **Трекинг экспериментов**
   - Провести серию экспериментов (RandomForest vs LogisticRegression).
   - Выбрать лучшую модель через UI MLflow.
